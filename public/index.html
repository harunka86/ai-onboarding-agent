AI Onboarding Agent

Rule-based customer onboarding pipeline — validate → compliance → risk → provision

Decisions

approved Passed all checks. Account provisioned, intro call scheduled.

escalated Validation failure or compliance flag. Routed to human review.

pending_review Risk score > 0.7. Awaiting human approval.

Endpoints

GET/api/onboard

Run all 10 sample customers. Returns a summary and full result array.

curl https://ai-onboarding-agent-pi.vercel.app/api/onboard
POST/api/onboard

Process a single customer. Send a JSON body with the fields below.

curl -X POST https://ai-onboarding-agent-pi.vercel.app/api/onboard \
  -H "Content-Type: application/json" \
  -d '{
    "id": "cust-001",
    "email": "alice@example.com",
    "company": "Acme Corp",
    "country": "US",
    "tier": "pro",
    "seats": 25
  }'

Required fields

id       string   Unique customer identifier
email    string   Contact email
company  string   Company name (used to infer industry)
country  string   ISO 3166-1 alpha-2 code (e.g. "US", "DE", "NG")
tier     string   "free" | "pro" | "enterprise"
seats    integer  Number of seats (≥ 1)
